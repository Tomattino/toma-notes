TypeScript + Vue 3 教材

型の基礎からVueとの組み合わせまで。Vue教材の内容を前提とする。

TypeScriptとは何か

TypeScript（TS）はJavaScriptに型を追加した言語。書いたコードはブラウザ実行前にJavaScriptに変換（トランスパイル）される。

型を書くことで「この変数には文字列しか入らない」「この関数は数値を返す」というルールをコードで表現できる。VSCodeが型情報を使ってリアルタイムにエラーを教えてくれる。

JavaScript — 実行するまでエラーに気づかない
function double(n) {
  return n * 2
}

double('hello')  // NaN になるが実行まで気づかない
TypeScript — 書いた瞬間にエラーがわかる
function double(n: number): number {
  return n * 2
}

double('hello')
// ↑ エラー: string は number に代入できない

オブジェクト型

// インライン
const user: { name: string; age: number } = { name: '山田', age: 32 }

// type エイリアス（名前をつけて再利用する）

  id:    number
  name:  string
  email: string
  age?:  number   // ? をつけるとオプション（省略可）
}

const user: User = { id: 1, name: '山田', email: 'a@example.com' }

// interface（オブジェクトの型定義。typeと似ているが継承できる）
interface Recipe {

}
typeinterface

ユニオン型 — 複数の型のどれか

type Status = 'active' | 'inactive' | 'pending'  // 文字列リテラルのユニオン
type ID     = number | string                       // 型のユニオン

let status: Status = 'active'
status = 'deleted'  // エラー: 'deleted' は Status 型にない

特殊な型

// any — 型チェックをオフにする（なるべく使わない）
const data: any = 'なんでも入る'

// unknown — anyより安全。使う前に型チェックが必要
const input: unknown = getUserInput()
if (typeof input === 'string') {
  console.log(input.toUpperCase())  // ここで初めて文字列として使える
}

// void — 戻り値がない関数
function log(msg: string): void {
  console.log(msg)
}

// never — 絶対に到達しない（無限ループ・必ずthrowする関数）
function throwError(msg: string): never {
  throw new Error(msg)
}

関数の型定義

// 引数と戻り値に型をつける
function add(a: number, b: number): number {
  return a + b
}

// アロー関数
const multiply = (a: number, b: number): number => a * b

// オプション引数とデフォルト値
function greet(name: string, greeting?: string): string {
  return `${greeting ?? 'こんにちは'}, ${name}`
}

// 非同期関数 — 戻り値はPromise<T>
async function fetchUser(id: number): Promise<User> {
  const res = await fetch(`/api/users/${id}`)
  return res.json()
}

ジェネリクス — 型を引数にする

「具体的な型は使う側が決める」という仕組み。汎用的な関数やコンポーネントで使う。

// T は型パラメータ。呼び出し時に具体的な型が決まる
function first<T>(arr: T[]): T | undefined {
  return arr[0]
}

first([1, 2, 3])         // 戻り値はnumber | undefined
first(['a', 'b'])        // 戻り値はstring | undefined

// API レスポンスをジェネリクスで表現する実務的な例
type ApiResponse<T> = {
  data:    T
  status:  number
  message: string
}

type UserResponse   = ApiResponse<User>
type RecipeResponse = ApiResponse<Recipe[]>

ref の型推論と明示的な型定義

// 型推論 — 初期値から自動で型が決まる

const name   = ref('')      // Ref<string>

// 明示的に型を指定する（初期値がnullやundefinedのとき必須）

Pinia の型定義

// src/stores/recipeStore.ts
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'
import type { Recipe } from '../types'  // 型のimportはimport typeで

export const useRecipeStore = defineStore('recipe', () => {
  // 型推論で十分なものはそのまま、nullを含む場合は明示的に指定
  const recipes      = ref<Recipe[]>([])
  const selectedKey  = ref<string>('')
  const isLoading    = ref(false)

  const selectedRecipe = computed(
    (): Recipe | undefined =>
      recipes.value.find(r => r.name === selectedKey.value)
  )

  const selectRecipe = (key: string): void => {
    selectedKey.value = key
  }

  const fetchRecipes = async (): Promise<void> => {
    isLoading.value = true
    try {
      const res = await fetch('/api/recipes')
      recipes.value = await res.json()
    } finally {
      isLoading.value = false
    }
  }

  return { recipes, selectedKey, isLoading, selectedRecipe, selectRecipe, fetchRecipes }
})

よく使う実践パターン

API通信のパターン（エラーハンドリング含む）

<script setup lang="ts">
import type { Recipe } from '../types'

const recipes  = ref<Recipe[]>([])
const isLoading = ref(false)
const error     = ref<string | null>(null)

onMounted(async () => {
  isLoading.value = true
  error.value = null
  try {
    const res = await fetch('/api/recipes')
    if (!res.ok) throw new Error(`HTTP error: ${res.status}`)
    recipes.value = await res.json() as Recipe[]
  } catch (e) {
    error.value = e instanceof Error ? e.message : '取得に失敗しました'
  } finally {
    isLoading.value = false
  }
})
</script>

<template>
  <div v-if="isLoading">読み込み中...</div>
  <div v-else-if="error">{{ error }}</div>
  <div v-else>
    <RecipeCard v-for="recipe in recipes" :key="recipe.name" :dish="recipe" />
  </div>
</template>

型ガード — 型を絞り込む

// unknownの型を絞り込む
function isRecipe(val: unknown): val is Recipe {
  return (
    typeof val === 'object' &&
    val !== null &&
    'name' in val &&
    'img' in val
  )
}

const data: unknown = fetchSomething()
if (isRecipe(data)) {
  console.log(data.name)  // ここでRecipe型として扱える
}

Refの型を別の変数に渡すとき

import type { Ref } from 'vue'

// 関数の引数にRefを受け取る場合
function useRecipeFilter(recipes: Ref<Recipe[]>, keyword: Ref<string>) {
  const filtered = computed(() =>
    recipes.value.filter(r => r.name.includes(keyword.value))
  )
  return { filtered }
}

よくあるミスと対処

ミス原因対処
Object is possibly 'null' null を含む型を使う前に型チェックしていない if (user.value !== null) でチェックするか、user.value! の非nullアサーション（確実なときのみ）
Property does not exist 型にないプロパティにアクセスしている 型定義にプロパティを追加するか、型が合っているか確認する
anyを使いすぎる 型をつけるのが面倒でanyで逃げる anyを使うと型チェックの恩恵がなくなる。unknownを使うか正しい型を定義する
型インポートで余分なコードが出る 型をimportするとき通常のimportを使っている import type { Recipe } を使う。型だけのimportであることを明示するとバンドルサイズが減る

実務チェックリスト

確認項目内容
型定義ファイルを src/types/ にまとめているか各ファイルにバラバラに書くと管理が難しくなる
any を使っていないか型がわからないときは unknown を使って型ガードで絞り込む
null を含む ref に型を明示しているかref<User | null>(null) のように明示する
型のimportは import type を使っているかバンドルサイズの削減と意図の明示
props の型定義に withDefaults を使っているかオプションのpropsにデフォルト値を設定する
API レスポンスに型をつけているかas Type でキャストするか、型ガードで安全に扱う
Pinia store の関数に戻り値の型をつけているか戻り値が明確になりコンポーネントから使いやすくなる
